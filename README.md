# NSPD Parser

Скрипт `parser.py` автоматизирует получение URL карточек объектов недвижимости на карте НСПД (https://nspd.gov.ru/map) по списку кадастровых номеров.  
Результат — JSON‑файл с соответствием `кадастровый номер → URL` с параметром `selectedCard`.

## Возможности

- Открывает карту НСПД в Chrome через Selenium.
- Обходит проблему с сертификатом (страница «Подключение не защищено»).
- Для каждого кадастрового номера:
  - находит поле «Поиск объектов недвижимости» внутри nested Shadow DOM;
  - вводит номер и запускает поиск;
  - выбирает первый (и единственный) результат в списке найденных объектов;
  - дожидается появления `selectedCard` в URL и сохраняет его.
- Записывает все результаты в `result.json`.

## Требования

- Python 3.10+ (рекомендуется 3.11).
- Google Chrome (актуальная стабильная версия).
- Установленные пакеты:
  - `selenium`
  - `webdriver-manager`

Установка зависимостей в виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -U pip
pip install selenium webdriver-manager
```

## Входные данные

Скрипт ожидает файл `cad_list.txt` в корне проекта.
Каждая строка — один кадастровый номер, без лишних пробелов и разделителей, например:

58:24:0381401:620
58:24:0381401:342

Пустые строки игнорируются.

## Выходные данные

После завершения работы создаётся файл result.json следующего вида:

[
  {
    "cad_num": "58:24:0381401:620",
    "url": "https://nspd.gov.ru/map?...&selectedCard=..."
  },
  {
    "cad_num": "58:24:0381401:342",
    "url": "https://nspd.gov.ru/map?...&selectedCard=..."
  }
]

Если для какого‑то номера произошла ошибка (элемент не найден, таймаут и т.п.), для него записывается "url": null.

## Как работает скрипт

**1. Запуск браузера**

Функция `start_driver()`:

поднимает Chrome через webdriver-manager;

настраивает окно (--window-size=1600,900);

отключает проверки сертификатов (--ignore-certificate-errors), что нужно для НСПД;

задаёт page_load_timeout(60).

**2. Открытие карты НСПД**

Функция open_map(driver):

Переходит по адресу `https://nspd.gov.ru/map`.

Ждёт появления тега <body>.

Если браузер показал страницу с ошибкой сертификата, автоматически нажимает «Дополнительно» → «Перейти».

Даёт приложению время загрузиться (time.sleep(8)).

Для диагностики выводит в консоль количество найденных элементов <m-sidebar>, <m-search-field> и <input>.

В текущей версии open_map(driver) вызывается перед каждым кадастровым номером, то есть карта заново загружается для каждого объекта. Это гарантирует стабильное состояние интерфейса, но замедляет работу. В дальнейшем можно оптимизировать, реализовав нажатие кнопки «Назад» внутри m-selected-object-page вместо полной перезагрузки карты.

**3. Поиск и выбор участка**
Функция `get_url_for_cad(driver, cad_num, timeout=90)` делает основную работу:

Находит хост m-sidebar в основном DOM и берёт его shadow_root.

Внутри shadow_root m-sidebar находит m-search-field и его shadow_root.

В shadow_root m-search-field ищет form > label.input-label > input — это поле «Поиск объектов недвижимости».

Очищает поле, вводит кадастровый номер и нажимает Enter.

В shadow_root m-sidebar ждёт появления m-found-objects (список найденных объектов), берёт его shadow_root.

Внутри shadow_root m-found-objects находит m-accordion и его shadow_root.

В shadow_root m-accordion выбирает первый button span (единственный результат) и кликает по нему.

Ждёт, пока в driver.current_url появится параметр selectedCard=....

Возвращает итоговый URL.

Вся работа с Shadow DOM выполняется через свойство element.shadow_root, доступное в Selenium 4.

**4. Основной цикл**
Функция `main()`:

Читает список кадастровых номеров из cad_list.txt.

Стартует Chrome через `start_driver()`.

Для каждого cad_num:

вызывает `open_map(driver)` (свежая загрузка карты);

вызывает `get_url_for_cad(driver, cad_num)` и добавляет результат в список;

пишет в консоль подробный лог шагов.

В конце записывает список результатов в result.json и закрывает браузер.

## Использование

`bash`
`python parser.py`
При запуске в консоль выводится подробный лог, по которому можно отладить проблемы (таймауты ожидания, изменения в DOM и т.п.).

## Планируемые улучшения

Вместо полной перезагрузки карты для каждого участка реализовать корректный возврат к поиску по кнопке «Назад» в m-selected-object-page внутри того же m-sidebar.

Добавить более точную обработку ошибок (например, различать «участок не найден» и «сайт не отвечает»).

Вынести переключатель тихого режима, таймауты и параметры Chrome в конфигурационный файл.
